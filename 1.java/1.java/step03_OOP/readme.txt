[1]. 학습내용

1. 상속
	- 클래스 간의 상속관계
	
2. 다형성
	- 상속이 전제인 상태의 구조 (상속이 안되어있으면 절대 쓸수 없음)

3. 주의 사항
	- jdk 1.5 부터 추가된 메카니즘은 정통 학습 직후에만 반영
[2] 학습 내용
1. 객체 생성 순서
 	object -> 부모 객체 -> 자식 객체 순으로 생성
2. 생각해보기
	1. id/pw 를 실제 보유했던 parent 객체가 생성될때 메모리에 사용가능하게 생성
	2. 하나씩 생성
	
3. 암기 사항
	1. java.lang.Object 최상위 root
	2. class 는 단일 class만 상속 가능
		단, 수직 관계의 계층 레벨 제한 없음
	3. ***상속 받는 구성 요소는 단 2가지*** ( static{} 과 생성자는 상속받지 않음 )
		1) 멤버변수
		2) 메소드
	4. 자식 생성자 호풀해서 실행시 내부적으로 조상, 부모, 자식 순으로 객체 생성
		- 생성자 내부에 super([..])가  argument가 일치되는 부모 생성자 호추
		- 미 코딩시 컴파일러가 자동 추가
	5. java 스펙의 협약
		- 자식 is a 부모 관계에서만 상속 형성
		( 매니저 is a 직원 ) : 상속 가능
		( 고객 is a 직원 ) : 성립 불가, 상속 관계 비추
	6. *** 상속받은 메소드 {} 로직 수정 가능  (재정의 =override)
	
4. 용어 정리 
	1. 동일한클래스에서 동일한 이름의 생성들 또는 메소드들 개발 가능
		단, argument는 다름
			- overloading (overload), 다중정의
	2. 상속 받은 자식 또는 자손 클래스에서 메소드 내용 수정
		 - overriding(override), 재정의
